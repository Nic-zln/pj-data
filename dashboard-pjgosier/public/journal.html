Phase 1 – Consolidation et structuration des données

14/05/2025 — Nicolas
Difficulté : Il a été difficile de canaliser les discussions du groupe, qui partaient souvent dans toutes les directions.
Solution : Mise en place d’une dictature plus rigoureuse, avec la désignation d’un seul référent de séance pour maintenir le fil conducteur.
Avis général : L’expérience a permis de mieux comprendre l’importance d’un cadre clair et strict pour que chacun travaille efficacement.
28/05/2025 — Saël
Difficulté : Import des données SIRENE : de nombreux champs étaient mal formatés.
Solution : Après plusieurs essais, nous avons développé un script Python pour nettoyer et normaliser les données.
Avis général : Bonne collaboration, chacun a proposé des idées pour avancer plus vite.
28/05/2025 — Nicolas
Difficulté : Beaucoup de doublons dans les données « personnes » (pages blanches).
Solution : Utilisation d'un algorithme de dédoublonnage basé sur le nom et l'adresse. Résultat : base beaucoup plus propre !
Avis général : Un peu de frustration au début, mais satisfaction après avoir vu le résultat.
28/05/2025 — Harry
Difficulté : Les données d'associations étaient incomplètes et parfois obsolètes.
Solution : Recherche de sources complémentaires (site du Gosier) et fusion des fichiers pour obtenir une table plus fiable.
Avis général : L'équipe a fait preuve de rigueur et d'une bonne collaboration pour surmonter les difficultés rencontrées lors du traitement des données.
Objectif de fin de séance : Avoir 3 tables homogènes, exploitables et documentées.
